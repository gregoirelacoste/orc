PHASE STRATÉGIE

Tu as fait ta recherche initiale. Transforme-la en plan d'action.

1. Lis .orc/research/INDEX.md et tous les SYNTHESIS.md
2. Relis .orc/BRIEF.md (la vision immuable)

Produis une .orc/ROADMAP.md structurée en EPICS :

```
## Epic 1 : [nom]
Justification : [référence vers un fichier .orc/research/...]
- [ ] Feature 1.1 — description | critères d'acceptance
- [ ] Feature 1.2 — description | critères d'acceptance

## Epic 2 : [nom]
Justification : [référence .orc/research/...]
- [ ] Feature 2.1 — ...
```

Règles :
- Chaque epic = 3-5 features liées thématiquement
- Chaque feature DOIT référencer un insight de .orc/research/
- Ordonne les epics par : impact utilisateur × faisabilité
- Les quick wins (haute valeur, faible effort) en premier
- Les features de fondation technique avant les features avancées
- Critères d'acceptance = comment vérifier que c'est fait

Mets à jour CLAUDE.md avec les patterns techniques
découverts pendant la recherche (APIs à utiliser, conventions du domaine, etc.)
